Hard truth

Finishing this module proves you can run the checklist — it does not auto-certify you. A lead reviews your sandbox ticket (kill criteria sharp enough to actually kill the bet) and graduates you to the live board.

SOP dry-run — what to produce

  1. Assumption — paste your Module 1 one-sentence statement (or refine it).
  2. Ticket — six sections with numeric pass/kill in Measure and placeholder Learn.
  3. Backlog — note which labels you would apply (experiment required).
  4. Build — smallest test described; note if you would use the skill chain.
  5. Measure — what you would post weekly; duration and kill threshold.
  6. Learn — write a sample decision (persevere / pivot / kill) as if data landed on kill day.
  7. Done — one line on what happens next per your decision.

You can capture steps 3–7 as comments on your sandbox issue or in a short “SOP walkthrough” section at the bottom.
