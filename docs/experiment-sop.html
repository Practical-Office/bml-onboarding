This is the full checklist for running a Build-Measure-Learn experiment on the live BML Board. The board, labels, and project are already set up — you orient in Module 3, then run from this list. Keep quick reference open for gates, WIP, and the ticket template.

Each course module teaches one slice of these steps. Module 7 packages your cert evidence (draft ticket + skill chain plan). Live board work starts in Homework F after lead approval.

Experiment SOP — seven steps

  1. State the risky assumption. One sentence: what must be true for the bet to work, and what breaks if it is wrong. Module 1 · SOP-1
  2. Draft the experiment ticket. Six sections: Hypothesis, Build (smallest test), Measure (numeric pass and kill + duration), Learn (placeholder until data), Acceptance Criteria, Technical Context. Module 4 · SOP-2
  3. Label and Backlog. Apply experiment (required). Add phase or decision labels as needed. Issue on Backlog until hypothesis + kill criteria are written. Modules 2–3 · SOP-3
  4. Build. Ship the smallest test. Non-trivial work: /grill-with-docs → /to-spec → /to-tickets → /implement. Tiny builds may skip to /implement. Module 4 · SOP-4
  5. Measure. Collect only the evidence you pre-registered. Post weekly numbers on the issue. No new product scope unless instrumentation is broken. Module 5 · SOP-5
  6. Learn / Validate. Write what you learned. Decide persevere, pivot, or kill (kill is success when criteria hit). Apply the matching decision label. Modules 2, 6 · SOP-6
  7. Done — record what’s next.
    • Pivot → revise hypothesis; restart at step 1.
    • Persevere → next bet or deeper build on same path.
    • Kill → stop the bet; archive learning so theater does not continue.
    Modules 6–7 · SOP-7

Hard truths

  • If Learn rarely produces kill, pivot, or persevere, the system is theater.
  • Max 3 active cards in Build + Measure combined.
  • If you have not killed something in 60 days, you failed the system.
