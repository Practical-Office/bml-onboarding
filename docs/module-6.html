Playbook

Why operating rules exist

If Learn does not regularly produce kill, pivot, or persevere decisions, you are not running Build Loop — you are performing it.

A board without gates becomes a feature factory with extra columns. WIP limits force finish-over-start. Weekly cadence prevents cards rotting in Learn. Anti-theater rules exist because teams naturally optimize for shipping — not for stopping bad bets.

Operating discipline is how a small team runs multiple experiments without drowning in half-finished Builds or endless Measure phases with no decision.

Ideas to hold

  • Columns are phases, not suggestions. Each transition has a gate. Skipping a gate erases auditability.
  • Labels mark decisions, not phases. Only experiment plus Learn outcomes (persevere, pivot, kill-candidate). No phase labels — columns own phase.
  • The issue body holds evidence. Board cards are pointers; numbers and decisions live on the issue.
  • Three active maximum. Build + Measure combined — for a team of three, that is one experiment per person at most.
  • Learn clears before new Builds. If Learn is stuck, fix that before opening more experiments.

How-to: column gates

From → ToGate (all required)Common miss
Backlog → Build One-sentence hypothesis; numeric kill criteria; duration set “We’ll figure out metrics during Build”
Build → Measure Thinnest test shipped; instrumentation or manual log path named Moving when code merged but nothing can be counted yet
Measure → Learn Duration complete; weekly numbers recorded on the issue Moving early because “it looks good so far”
Learn → Done Decision label applied; evidence + one next action written Card moved to Done with no label or follow-up

How-to: 3-active WIP limit

Count every card in Build or Measure. Maximum three at once for a three-person team.

  1. Before pulling from Backlog, count active cards. At three? Finish or Learn something first.
  2. Measure counts as active — it consumes attention even when no code ships.
  3. Long-running Measure is not a license for parallel Builds. Either extend duration explicitly on the issue or move to Learn with an interim decision.
  4. Backlog can be deep. Active must stay shallow.

How-to: weekly cadence (~30 min total)

WhenRitualOutput
Monday Board review: unblock Measure; post weekly numbers; move anything past duration into Learn Every active experiment has fresh numbers or a named blocker
Wednesday Scope check: any silent Build creep on active cards? Creep either rejected or spun into a new Backlog experiment
Friday Learn sweep: every Learn card gets a decision or explicit defer with a date Zero orphan Learn cards without a plan

Learn-phase decisions

LabelMeaningTypical next action

Micro-scenario: stuck Learn queue

Abstract ops story — no product names.

Three experiments sit in Learn. Two have complete numbers; one is waiting on “one more week” for the third time. Meanwhile, Backlog has five eager ideas and someone wants to start two new Builds.

Anti-theater response:

  1. Friday Learn sweep: decide the two complete cards (one persevere, one kill-candidate).
  2. Defer card three with a date, or kill it — “waiting for perfect data” is not a decision.
  3. Hold new Builds until Learn count drops. Pull one Backlog card only after WIP drops below three.
  4. Post mortem note on the issue: “Learn queue stuck 3 weeks — weekly Friday sweep now mandatory.”

Week-1 Measure can be a spreadsheet. Event IDs should match the metrics dictionary so later instrumentation does not invent new names.

Failure modes — Learn anti-theater

  • Zero kills in a quarter. Either your kill criteria are too soft or Learn is being skipped. Fix Learn before more Builds.
  • Learn without numbers. Decisions based on memory or enthusiasm. Gate: metrics on the issue.
  • Perpetual Measure. Extending duration informally to avoid a kill. Update the issue or decide.
  • Board as wallpaper. Team ships from memory; cards do not move. Monday review is non-optional.
  • WIP blowout. Six active experiments, none reaching Learn. Enforce the three-active cap ruthlessly.
  • Label without action. pivot applied but no follow-up experiment filed. Done means done and next step exists.
